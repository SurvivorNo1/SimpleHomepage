名词解释

P1

1.语义网络

语义网络是一种采用网络形式表示人类知识的方法。一个语义网络是一个带标识的有向图。其中，带有标识的结点表示问题领域中的物体、概念、事件、动作或者态势。

2.上下文无关文法

又称2型文法。在上下文无关文法中，每条产生式规则的左侧必须是一个单独的非终结符。每一条规则都采用A→x 的形式，其中A∈N,x∈V∗ 。在这种体系中，规则被应用时不依赖于符号A所处的上下文。

3.专家系统

专家系统是一种智能的计算机程序，它运用知识和推理来解决只有专家才能解决的复杂问题。

专家系统是一种具有特定领域内大量知识与经验的程序系统，它应用人工智能技术、模拟人类专家求解问题的思维过程求解领域内的各种问题，其水平可以达到甚至超过人类专家的水平。

4.过拟合

过拟合是指为了得到一致假设而使假设变得过度严格。表现为模型在训练集上表现很好，但在测试集上却表现很差。

5.归结反演

应用归纳定理证明的过程称之为归结反演，又称为消解反演.

P2

1.随机梯度下降

随机梯度下降是一种通过迭代的方式来最小化损失函数的优化算法。其主要思想是通过不断调整模型参数，使得损失函数达到最小值。与传统的梯度下降算法不同，随机梯度下降每次更新参数时仅使用一个训练样本（或者一个小批量的样本），通过计算他们梯度的平均值来更新梯度, 而不是整个数据集。

2.上下文无关文法

又称2型文法。在上下文无关文法中，每条产生式规则的左侧必须是一个单独的非终结符。每一条规则都采用A→x 的形式，其中A∈N,x∈V∗ 。在这种体系中，规则被应用时不依赖于符号A所处的上下文。

3.知识图谱

又称科学知识图谱，用各种不同的图形等可视化技术描述知识资源及其载体，挖掘、分析、构建、绘制和显示知识及它们之间的相互联系。它把复杂的知识领域通过数据挖掘、信息处理、知识计量和图形绘制而显示出来，揭示知识领域的动态发展规律。本质是一种大规模语义网络。

4.过拟合

过拟合是指为了得到一致假设而使假设变得过度严格。表现为模型在训练集上表现很好，但在测试集上却表现很差。

5.图灵测试

图灵提出的一种测试机器是不是具有人类智能的方法：一个人在不接触对方的情况下，通过一种特殊的方式，和对方进行一系列的问答，如果在相当长时间内，他无法根据这些问题判断对方是人还是计算机，那么，就可以认为这个计算机具有同人相当的智力，即这台计算机是能思维的。

1.强化学习中的value-based、policy-based代表算法有？区别是什么

其中policy based包括TRPO、PPO算法，value based包括DQN、DDQN、DDPG和TD3

Policy-Based的方法直接输出下一步动作的概率，根据概率来选取动作。但不一定概率最高就会选择该动作，还是会从整体进行考虑。适用于非连续和连续的动作。常见的方法有Policy gradients。

Value-Based的方法输出的是动作的价值，选择价值最高的动作。适用于非连续的动作。常见的方法有Q-learning、Deep Q Network和Sarsa。

2.结合A算法，说明A*算法流程，总结其特点

P176、177

  • 初始化open_set和close_set；

  • 将起点加入open_set中，并设置优先级为0（优先级最高）；

  • 如果open_set不为空，则从open_set中选取优先级最高的节点n：

    • 如果节点n为终点，则：

      • 从终点开始逐步追踪parent节点，一直达到起点；

      • 返回找到的结果路径，算法结束；

    • 如果节点n不是终点，则：

      • 将节点n从open_set中删除，并加入close_set中；

      • 遍历节点n所有的邻近节点：

        • 如果邻近节点m在close_set中，则：

          • 跳过，选取下一个邻近节点

        • 如果邻近节点m也不在open_set中，则：

          • 设置节点m的parent为节点n

          • 计算节点m的优先级

          • 将节点m加入open_set中

1.强化学习的奖励函数设计存在什么问题？为什么深度Q网络需要采用价值函数近似的表示方法。

问题

  1. 稀疏奖励问题： 智能体只在完成任务的特定阶段或达到目标时才获得奖励，导致学习过程变得困难。

  2. 奖励稀疏性和延迟问题： 智能体可能需要等到完成整个任务后才能收到奖励，导致学习延迟和不稳定性。

  3. 超越目标问题： 简单或不完善的奖励函数可能导致智能体找到不符合实际需求的策略。

  4. 奖励偏差问题： 过于激励某些行为可能使智能体忽略其他重要方面。

  5. 不确定性问题： 环境中的不确定性和噪声可能使奖励信号不准确。

  6. 人类先验知识问题： 基于人类先验知识设计奖励函数可能难以获取或不准确。

原因

DQN采用价值函数近似的表示方法是为了处理高维状态空间，通过使用深度神经网络来估计Q函数，以便在复杂环境中学习和推广对动作值的估计，同时减少对大量状态的直接存储需求。

2.讨论NLP技术的瓶颈，说说你对CHATGPT核心技术的理解。

NLP技术瓶颈讨论：

  1. 理解上下文： 尽管最先进的NLP模型在理解上下文方面取得了显著进展，但仍存在难以捉摸的语境、歧义以及对话中的长期依赖问题。模型在处理复杂上下文时可能会失去准确性。

  2. 常识推理： 让模型具备常识推理和背景知识仍然是一个挑战。一些情境中，模型可能在缺乏先验知识时做出不合理或不准确的推断。

  3. 生成文本的一致性和合理性： 生成式模型在处理开放领域对话时可能会产生不合理或矛盾的回应，缺乏对生成文本一致性的有效控制。

  4. 数据偏见： 模型在训练过程中可能从大规模的文本数据中学到不良的偏见，并在生成文本时反映这些偏见。解决这一问题需要更好的数据清理和平衡。

  5. 领域特定性： 大部分预训练模型对特定领域的适应性较差，对于专业领域或特定行业的语言理解和生成任务，需要更精细的调整和训练。

  6. 资源需求： 训练和部署大型NLP模型需要大量的计算资源，这对于一些小规模或资源有限的应用场景来说可能是一个瓶颈。

CHATGPT核心技术理解：

  1. Transformer架构： CHATGPT采用Transformer架构，充分利用自注意力机制，使其能够在处理长文本时更好地捕捉上下文信息。

  2. 大规模预训练： CHATGPT通过大规模文本数据的预训练，使模型具备广泛的语言知识和语境理解，为后续生成式任务提供强大的基础。

  3. 对话式交互： CHATGPT专注于对话式交互，通过训练来处理用户输入，并生成连贯、上下文合理的回应。模型被设计成适应多样的对话情境。

  4. 人类反馈的强化学习： 使用强化学习方法有助于提高生成文本的质量和适用性，使模型更好地满足用户需求。

  5. 多模态输入： CHATGPT不仅仅限于纯文本输入，还能够处理多模态输入，包括图像、描述等，增加了模型的应用领域和适应性。

  6. 训练监督策略模型、训练奖励模型（Reward Model，RM）、采用PPO（Proximal Policy Optimization，近端策略优化）强化学习来优化策略。
